# Next.js Frontend with AWS Amplify

This is the Next.js frontend application connected to the AWS Amplify backend.

## Setup

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Ensure amplify_outputs.json is present:**
   The `amplify_outputs.json` file should be in the `public/` directory. This file is generated by the Amplify backend deployment and contains all the configuration needed to connect to your backend services.

3. **Start the development server:**
   ```bash
   npm run dev
   ```

## Features

- **Todo Management**: Full CRUD operations for todos using AWS AppSync and DynamoDB
- **Authentication Ready**: Configured for AWS Cognito (can be enabled when needed)
- **Type-Safe**: TypeScript types generated from your Amplify backend schema
- **Modern UI**: Built with shadcn/ui components and Tailwind CSS

## Backend Connection

The frontend connects to the Amplify backend through:

1. **Configuration**: `lib/amplify-config.ts` loads `public/amplify_outputs.json`
2. **Data Client**: Uses `generateClient<Schema>()` from `aws-amplify/data`
3. **Types**: TypeScript types are imported from `@/amplify/data/resource`

## Available Scripts

- `npm run dev` - Start development server
- `npm run build` - Build for production
- `npm run start` - Start production server

## Environment Variables

No environment variables are required for local development. The `amplify_outputs.json` file contains all necessary configuration.

For production deployments, Amplify will automatically inject the outputs during the build process.
